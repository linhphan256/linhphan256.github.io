Setting up Academic page in Linux with Wowchemy

How I setup this website using Wowchemy, Github pages, Linux

Linh T Phan
Dec 25, 2021 2 min read Demo
Image credit: Unsplash

Overview

Since everyone has website, I record how I build this website. This post is heavily influened by the post of Pramod but without the need of using root

Setup Golang and Hugo

The pre-requisite for building this blog is Hugo framework but Hugo requires Golang so in this 1st step, we need to install Golang before anything else

To install Golang, please check its homepage and download the lastest release of Golang.

  1. Extract the compressed file.
  2. Suppose the folder is located at ~/code/go1.17.5.linux-amd64, we need to add the following line to .zshrc or .bashrc so your terminal can find go when it needs
    export PATH=$PATH:~/code/go1.17.5.linux-amd64/bin
  1. Rehash your $PATH
    source .zshrc
  1. Check if your system can find go by typing the following in your terminal
    go version

Then, we need to install Hugo

  1. Clone source of Hugo to your local machine by
git clone https://github.com/gohugoio/hugo.git
  1. Go to your clone folder and install Hugo
cd hugo
CGO_ENABLED=1 go install -v --tags extended
  1. The installation creates a folder under your home named go. Copy the hugo binary file to the location of your go
cd ~/go/bin
cp hugo ~/code/go1.17.5.linux-amd64/bin

Test hugo by

    hugo version

Academic template

Clone the source of Academic template

git clone https://github.com/wowchemy/starter-academic.git

To run the default version of template, try this

cd ~/path-to-starter-academic/
hugo server

The following error will occur

Error: Error building site: failed to render pages: render of "term" failed: execute of template failed: template: _default/list.html:5:3: executing "_default/list.html" at <partial "site_head" .>: error calling partial: "/tmp/hugo_cache/modules/filecache/modules/pkg/mod/github.com/wowchemy/wowchemy-hugo-modules/wowchemy/v5@v5.0.0-20211122231030-d26f01d5d6c3/layouts/partials/site_head.html:208:13": execute of template failed: template: partials/site_head.html:208:13: executing "partials/site_head.html" at <getenv "WC_POST_CSS">: error calling getenv: access denied: "WC_POST_CSS" is not whitelisted in policy "security.funcs.getenv"; the current security configuration is:

Dont worry. Copy the error to search engine and it will point you to the solution on Stackoverflow. Basicly, just copy the following code to your path-to-starter-academic/config/_default/config.yaml. For me, I paste after module import

security:
  funcs:
    getenv:
      - ^HUGO_
      - ^WC_

Now check hugo server again. Everything works now :)

$ cd path-to-starter-acamedic
$ hugo server
